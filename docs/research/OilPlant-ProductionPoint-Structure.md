# Oil Plant ProductionPoint Structure Notes

These notes come from the user's old CBD/oil plant XML and matching `.i3d` file. The purpose is to learn the structure pattern, not to copy the asset into this mod.

## Important Rule

Do not upload the old oil plant `.i3d` or copied GIANTS/base-game assets into this repository.

Use this only as a reference for how the XML and i3d node names line up.


## Why We Cannot Blindly Copy The XML

The XML references exact i3d node paths such as:

```text
0>10|0|0
0>11|0
0>12|0
```

Those paths only make sense inside that oil plant `.i3d` hierarchy.

If we paste those mappings into the Hemp Greenhouse XML without creating the same node hierarchy in the Green Horizon greenhouse `.i3d`, FS25 will likely throw missing-node or invalid-placeable errors.

## What We Should Copy Conceptually

For the Green Horizon greenhouse i3d, we should create our own nodes with similar names/purposes:

```text
levelArea1Start / Width / Height
clearArea1Start / Width / Height
indoorArea1Start / Width / Height
testArea1Start / End
tipOcclusionUpdateArea1Start / End
infoTrigger
sellingStation or unload/input trigger group
unloadTrigger
unloadTriggerMarker
unloadTriggerAINode
playerTrigger
playerTriggerMarker
teleportNode
palletSpawner
spawnSpace1start / end
warningStripes_loadUnloadArea
warningStripes_palletSpawnArea
```

Then the Hemp Greenhouse XML should reference the node paths generated by the real Green Horizon `.i3d`, not the oil plant paths.
